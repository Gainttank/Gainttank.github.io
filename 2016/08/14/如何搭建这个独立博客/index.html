如何搭建这个独立博客

发表于 2016-08-13   |   分类于 教程   |  

总的流程：

1.下载安装Git,Node.js；
2.注册coding账号或github账号（在国内推荐使用coding，一下均以coding为例）；
3.在coding创建项目
4.添加ssh key到coding
5.安装hexo
6.使用hexo搭建，配置，并上传项目

下载并安装

  1、Git
  2、node.js

注册coding并创建一个项目

配置SSH keys：

以下教程主要参考CNFeat的《如何搭建一个独立博客–简明Github Pages与Hexo教程》

一、打开Git bash：

  方法一：开始菜单（若生成桌面快捷键也可直接快捷键）

  方法二：右击鼠标，在当前文件夹打开Git bash

二、配置SSH keys：

1.检查SSH keys的设置

如果提示：No such file or directory说明你是第一次使用git。

2.生成新的SSH keys：

然后系统会要你输入密码：

在回车中会提示你输入一个密码，这个密码会在你提交项目的时候使用，如果为空的话提交项目时则不用输入密码。这个设置是防止别人往你的项目里提交内容。

注意：输入密码时没有*字样，直接继续输入即可。

最后看到这样的界面，就成功设置ssh key了：

3.添加ssh key到coding

  • 打开ssh key保存的路径（图片中所指示的路径，是你自己的电脑上的）

  • 打开.ssh文件夹中的id_rsa.pub文件（用记事本打开即可），并复制里面所有内容

  • 浏览器打开coding并登陆自己的账号。

  • 将公钥复制进去，点击添加。OK

  • 测试是否添加成功:

注意：配置ssh key到coding也可以根据这个页面的内容进行操作

安装hexo：

建议根据hexo官方文档进行操作

打开Git bash,安装hexo：

建议安装hexo-sever

建站：

转到自己希望站点存放的目录地址，新建文件夹并重命名。

右击鼠标，git bash here。

文件夹中生成这些文件：

配置hexo：

1.由于coding.net需要名为Staticfile的文件作为静态文件部署的标志，所以：

2.打开配置文件_config.yml

此时就算是建好了。

3.可以在本地进行测试。

可简写为 $ hexo g，用于生成静态文件

可简写为 $ hexo s，用于启动服务器；默认访问网址为http://localhost:4000/
若端口被占用，可修改 $ hexo s -p 5000 此时，访问网址为http://localhost:5000/

5.若要写新文章，$ hexo new 新文章的名字 即可。

本地显示无误之后，就要将文章上传到项目上了。

若出现这样的问题：

再次上传即可成功。建议使用$ hexo deploy -g这条命令进行上传。

来到coding.net的自己的项目页，发现自己的东西上传成功。

6.转到Pages服务：

将部署分支修改为master，点击“立即开启”即可。

现在你就可以通过coding提供给你的链接访问自己的博客了。

注意：在前面的_config.yml文件的配置中，你想要用什么域名访问，就必须填该域名。

修改主题：

本人使用的主题是NexT,比较简约，个人比较喜欢。具体操作只要根据其官网上的流程走就OK。其中也有一些如多说评论，百度统计等第三方服务的使用教程，比较详细，可以借鉴。

404页面：

推荐使用腾讯公益404页面。
